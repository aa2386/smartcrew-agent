# RAG 系统评估指标

## 评估维度

RAG 系统需要从多个维度进行评估：

### 1. 检索质量

| 指标 | 计算方式 | 说明 |
|------|----------|------|
| 准确率 | 正确结果数 / 返回结果数 | 返回结果中有多少是相关的 |
| 召回率 | 正确结果数 / 总相关结果数 | 相关结果中有多少被返回 |
| F1 分数 | 2 * P * R / (P + R) | 准确率和召回率的调和平均 |
| MRR | 相关文档排名倒数平均 | 衡量排序质量 |

### 2. 生成质量

| 指标 | 说明 |
|------|------|
| 忠实度 | 回答是否基于检索内容 |
| 相关性 | 回答是否切题 |
| 连贯性 | 回答是否通顺 |
| 完整性 | 回答是否完整 |

### 3. 系统性能

| 指标 | 说明 |
|------|------|
| 检索延迟 | 向量检索耗时 |
| 生成延迟 | LLM 生成耗时 |
| 端到端延迟 | 完整流程耗时 |

## 评估方法

### 人工评估

- 准备测试问题集
- 人工标注预期答案
- 对比系统输出与预期

### 自动评估

- 使用 LLM 评估回答质量
- 使用基准数据集测试
- A/B 测试对比

## SmartCrew 评估建议

1. 建立标准测试集
2. 定期运行评估脚本
3. 记录评估结果趋势
4. 根据评估结果优化参数
