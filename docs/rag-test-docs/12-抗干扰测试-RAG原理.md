# RAG 技术原理详解

## 什么是 RAG

RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合信息检索和文本生成的技术架构。RAG 让大模型在回答问题之前，先检索相关文档，然后基于检索到的内容生成回答。

## RAG 核心流程

### 离线索引阶段

```
文档 → 文档加载 → 文档分割 → 向量化 → 存入向量数据库
```

### 在线检索阶段

```
用户问题 → 向量化 → 向量检索 → 获取相关文档 → 构建提示词 → LLM 生成回答
```

## RAG 核心组件

### 1. 文档加载器

将不同格式的文档加载为统一的文档对象。支持：
- 文本文件（.txt, .md）
- PDF 文档
- Word 文档
- HTML 网页

### 2. 文档分割器

将长文档切分成适当大小的片段：
- 按字符数分割
- 按段落分割
- 按句子分割

### 3. 嵌入模型

将文本转换为向量，使计算机能够理解文本的语义相似性。常用模型：
- text-embedding-v3（阿里云）
- text-embedding-ada-002（OpenAI）

### 4. 向量存储

存储文档片段的向量，支持相似性检索。常用数据库：
- Chroma
- Milvus
- PgVector

## RAG 优势

| 优势 | 说明 |
|------|------|
| 知识可更新 | 无需重新训练模型 |
| 可追溯来源 | 可以引用原始文档 |
| 成本较低 | 相比微调成本更低 |
| 减少幻觉 | 基于真实文档回答 |

## RAG vs 微调

| 方案 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| RAG | 需要实时更新知识 | 知识可更新、成本较低 | 需要维护知识库 |
| 微调 | 需要改变模型行为 | 模型行为可定制 | 成本高、不易更新 |

## SmartCrew 中的 RAG 实现

SmartCrew 使用 LangChain4j 框架实现 RAG 功能，支持：
- 多种文档格式解析
- 灵活的文档分割策略
- 阿里云 DashScope 嵌入模型
- Chroma 向量数据库
