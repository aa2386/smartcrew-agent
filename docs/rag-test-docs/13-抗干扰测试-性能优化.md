# RAG 系统性能优化

## 性能瓶颈

RAG 系统常见的性能瓶颈：
- 文档解析速度慢
- 向量化计算耗时
- 向量检索延迟高
- 上下文构建效率低

## 优化策略

### 1. 文档预处理优化

- 预处理文档，提前完成分割
- 使用异步处理队列
- 缓存常用文档片段

### 2. 向量化优化

- 批量处理嵌入请求
- 使用本地嵌入模型
- 缓存嵌入结果

### 3. 检索优化

- 使用高效的向量索引
- 预热常用查询
- 实现多级缓存

### 4. 提示词优化

- 限制上下文长度
- 使用摘要代替原文
- 动态调整 top-k 参数

## 监控指标

| 指标 | 说明 | 目标值 |
|------|------|--------|
| 检索延迟 | 向量检索耗时 | < 100ms |
| 端到端延迟 | 完整 RAG 流程耗时 | < 3s |
| 召回率 | 相关文档召回比例 | > 90% |
| 准确率 | 检索结果准确比例 | > 85% |

## SmartCrew RAG 性能配置

```yaml
smartcrew:
  rag:
    enabled: true
    document:
      splitter:
        max-chunk-size: 200
        overlap-size: 50
    vector-store:
      type: chroma
      chroma:
        timeout-seconds: 60
```
